## Objetivos Definidos

## Pontos de Discussão Importantes

### 1. Gestão de Pessoas e Ponto
- **Registro Offline:** Necessidade crítica de serviço offline para registro de folha de ponto em obras rurais sem internet
- **Identificação:** Muitos colaboradores sem matrícula formal; necessário definir método eficaz de identificação
- **Tipos de Colaboradores:** Distinção entre CLT (motoristas, operadores) e diaristas (operários, carregadores)
- **Diaristas:** Sem contrato formal; sugestão de gerar contratos via sistema
- **Funcionalidades Futuras:** Banco de horas, gestão de férias, escalas (baixa prioridade inicial)

### 2. Controle de Ativos e Insumos
- **EPIs:** Controle rigoroso de entrada/saída por regulações de segurança
- **Equipamentos Caros:** Preocupação com entrega a diaristas; implementar notificações de não-devolução
- **Gestão de Materiais:** Controle manual via planilhas; app deve focar fluxo físico, não financeiro

### 3. Requisitos do Sistema e Usabilidade
- Sistema extremamente intuitivo para perfil variado de usuários
- Suporte a múltiplas empresas com dados unificados mas separados
- Interesse futuro em visão computacional para leitura de notas fiscais

## O que foi Definido

- Foco inicial em gestão de pessoas, ponto e controle de ativos
- Sistema deve funcionar offline para zonas rurais
- Distinção clara entre CLT e diaristas
- Prioridade: fluxo físico, não relatórios financeiros
- Primeiramente serão analisados os requisitos e tentaremos levar um protótipo de alta fidelidade para algum dos sistemas.
